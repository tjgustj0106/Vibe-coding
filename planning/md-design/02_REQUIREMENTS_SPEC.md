# Requirements Spec

## 1. Actors

| Actor | Description |
|---|---|
| Primary User | 하루 할 일·과제·일정을 등록하고 관리하는 대학생 |
| Optional Admin | 이번 MVP에서는 제외 — 단일 사용자 기준으로 처리 |

---

## 2. Main Use Cases

### UC-001. 할 일 / 과제 등록

- **Actor:** Primary User
- **Goal:** 할 일을 제목·상세내용·과목명·마감일과 함께 등록한다
- **Precondition:** 앱이 열려 있고 등록 폼에 접근할 수 있다
- **Main Flow:**
  1. 사용자가 "추가" 버튼을 누른다
  2. 제목(필수), 상세내용(선택), 과목명(선택), 마감일(선택)을 입력한다
  3. 확인 버튼을 누르면 항목이 저장된다
  4. 마감일이 지정된 경우, 오늘부터 마감일까지 매일 하루 뷰 할 일 목록에 자동 노출된다
- **Alternative Flow:**
  - 제목을 입력하지 않으면 저장되지 않고 오류 메시지가 표시된다
- **Result:** 새 항목이 하루 뷰 목록에 표시되고, 마감일이 있는 경우 월간 달력에도 표시된다

---

### UC-002. 하루 뷰 확인

- **Actor:** Primary User
- **Goal:** 오늘 해야 할 일 목록을 한눈에 확인한다
- **Precondition:** 등록된 항목이 하나 이상 있다
- **Main Flow:**
  1. 사용자가 앱을 열면 기본 화면이 오늘 날짜의 하루 뷰로 표시된다
  2. 오늘 날짜에 해당하는 할 일 목록이 표시된다 (마감일 자동 반영 포함)
  3. 각 항목의 제목과 완료 여부(체크박스)가 표시된다
  4. 항목을 클릭하면 상세 내용 모달이 열린다
- **Alternative Flow:**
  - 오늘 할 일이 없으면 "오늘 할 일이 없습니다" 안내 문구가 표시된다
- **Result:** 사용자가 오늘의 할 일을 빠르게 파악할 수 있다

---

### UC-003. 할 일 완료 체크 (상태 변경)

- **Actor:** Primary User
- **Goal:** 완료한 항목을 체크하여 진행 상황을 추적한다
- **Precondition:** 하루 뷰에 할 일 항목이 표시되어 있다
- **Main Flow:**
  1. 사용자가 완료한 항목의 체크박스를 클릭한다
  2. 항목이 완료 상태로 변경되고 시각적으로 구분된다 (취소선 또는 색상 변경)
  3. 상태는 localStorage에 저장된다
- **Alternative Flow:**
  - 이미 완료 상태인 항목의 체크박스를 다시 클릭하면 미완료 상태로 되돌아간다
- **Result:** 항목의 완료 상태가 저장되고 화면에 반영된다

---

### UC-004. 월간 달력 조회

- **Actor:** Primary User
- **Goal:** 한 달 전체의 일정과 과제 마감일을 달력 형태로 확인한다
- **Precondition:** 등록된 일정 또는 마감일이 있는 과제가 하나 이상 있다
- **Main Flow:**
  1. 사용자가 월간 뷰 탭을 선택한다
  2. 현재 월의 달력이 표시되고, 각 날짜에 해당 일정/마감 항목이 표시된다
  3. 날짜를 클릭하면 해당 날의 하루 뷰로 이동한다
  4. 이전/다음 월 이동 버튼으로 다른 달을 조회할 수 있다
- **Alternative Flow:**
  - 해당 월에 일정이 없으면 날짜에 아무 표시도 없이 빈 달력만 표시된다
- **Result:** 사용자가 한 달 단위로 일정 밀도와 마감일을 파악할 수 있다

---

### UC-005. 할 일 상세 내용 확인 및 수정/삭제

- **Actor:** Primary User
- **Goal:** 짧게 적힌 항목의 전체 내용을 확인하고 필요 시 수정하거나 삭제한다
- **Precondition:** 하루 뷰에 항목이 표시되어 있다
- **Main Flow:**
  1. 사용자가 항목 제목을 클릭한다
  2. 상세 내용 모달이 열리고 제목, 과목명, 상세내용, 마감일이 표시된다
  3. 수정 버튼을 누르면 편집 모드로 전환되어 내용을 변경할 수 있다
  4. 삭제 버튼을 누르면 확인 후 항목이 삭제된다
- **Alternative Flow:**
  - 수정 중 취소 버튼을 누르면 변경사항이 저장되지 않는다
- **Result:** 상세 내용이 저장·반영되거나 항목이 삭제된다

## 3. Functional Requirements

| ID | Requirement | Priority |
|---|---|---|
| FR-001 | 사용자는 제목(필수)·상세내용·과목명·마감일을 포함한 할 일 항목을 생성할 수 있다 | Must |
| FR-002 | 사용자는 오늘 날짜 기준의 하루 뷰에서 할 일 목록을 확인할 수 있다 | Must |
| FR-003 | 사용자는 월간 달력 뷰에서 일정과 마감일을 확인할 수 있다 | Must |
| FR-004 | 마감일이 지정된 항목은 오늘부터 마감 당일까지 하루 뷰에 자동으로 반영된다 | Must |
| FR-005 | 사용자는 할 일 항목의 완료 여부를 체크박스로 변경할 수 있다 | Must |
| FR-006 | 사용자는 항목 제목을 클릭하여 상세 내용 모달을 열 수 있다 | Must |
| FR-007 | 데이터는 로그인 없이 브라우저 localStorage에 저장·유지된다 | Must |
| FR-008 | 사용자는 항목을 수정하거나 삭제할 수 있다 | Should |
| FR-009 | 월간 달력에서 날짜를 클릭하면 해당 날의 하루 뷰로 이동한다 | Should |
| FR-010 | ~~앱은 항상 바탕화면 위에 표시되는 형태(always-on-top)로 사용할 수 있다~~ | **제외** — 웹 앱으로 구현 불가, Electron 전환 시 MVP 범위 초과 |
| FR-011 | 사용자는 완료 / 미완료 상태로 항목을 필터링할 수 있다 | Should |
| FR-012 | ~~사용자는 과목명으로 항목을 필터링할 수 있다~~ | **제외** — FilterTabs 복잡도 증가 대비 가치 낮음 |

## 4. Non-functional Requirements

| ID | Requirement |
|---|---|
| NFR-001 | 핵심 기능(할 일 추가·체크)을 처음 사용하는 사용자도 1분 안에 사용할 수 있어야 한다 |
| NFR-002 | 버튼과 입력 필드는 접근 가능한 이름(aria-label 등)을 가져야 한다 |
| NFR-003 | 민감한 정보는 GitHub 저장소에 커밋하지 않는다 |
| NFR-004 | 기본 MVP는 단일 사용자 기준으로 구현한다 (서버 없음, localStorage 사용) |
| NFR-005 | 화면 너비 1280px 이상 데스크탑 환경에서 정상 동작해야 한다 |
| NFR-006 | localStorage 데이터 구조는 향후 서버 연동을 고려한 JSON 형태로 설계한다 |

## 5. Acceptance Criteria

### AC-001. 할 일 등록

Given 사용자가 제목을 입력했을 때  
When 추가 버튼을 누르면  
Then 새 항목이 하루 뷰 목록에 표시된다

### AC-002. 과제 자동 반영

Given 사용자가 과목명과 마감일을 포함한 항목을 등록했을 때  
When 마감일 이전 날짜의 하루 뷰를 열면  
Then 해당 항목이 "[과목명] 과제" 형태로 목록에 자동 표시된다

### AC-003. 완료 체크

Given 하루 뷰에 미완료 항목이 있을 때  
When 체크박스를 클릭하면  
Then 항목이 완료 상태로 변경되고 시각적으로 구분된다

### AC-004. 상세 내용 확인

Given 하루 뷰에 항목이 있을 때  
When 항목 제목을 클릭하면  
Then 상세 내용 모달이 열리고 제목·과목명·상세내용·마감일이 표시된다

### AC-005. 월간 달력 조회

Given 마감일이 있는 항목이 등록되어 있을 때  
When 사용자가 월간 뷰를 열면  
Then 마감일에 해당 항목이 달력에 표시된다

### AC-006. 필터

Given 완료·미완료 항목이 섞여 있을 때  
When 완료 필터를 선택하면  
Then 완료된 항목만 표시된다
